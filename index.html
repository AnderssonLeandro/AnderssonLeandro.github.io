Clases y Objetos

Una clase es una plantilla o modelo para crear objetos que tienen características y comportamientos específicos. Un objeto es una instancia de una clase, y tiene su propia copia de los datos (atributos) y las funciones (métodos) definidos en la clase.

Interfaces

Garantiza que una clase implemente un conjunto específico de comportamientos, y es una forma de abstracción que se utiliza para mejorar la reutilización de código, la seguridad y la flexibilidad en el diseño de software.

Requerimientos Básicos del Sistema

El proyecto se encargará de controlar el ingreso, permanencia y salida de cada automóvil, para esto es necesario el módulo que se encargará de las gestión y control total de la información de las personas que se designen como usuarios del sistema.

Descripción del problema

• Cada cliente ingresa en un vehículo.
• El operario o empleado registra el vehículo, ingresa los datos del cliente y entrega un ticket.
• El cliente cuando va a retirar su vehículo solicita una cuenta de cobro y entrega el ticket.
• El operario genera los reportes de vehículos y también genera los reportes de los recaudos.

Conclusiones

Esta experiencia demuestra la capacidad del paradigma de programación orientada a objetos para ordenar de manera eficiente los requerimientos del problema de los parqueaderos, ha permitido establecer el tipo de objetos o instancias sus métodos y también las relaciones que se generan entre las clases del diagrama UML.
